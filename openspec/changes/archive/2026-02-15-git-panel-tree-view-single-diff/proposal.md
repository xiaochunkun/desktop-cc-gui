# Proposal: Git 面板新增树形视图与单文件 Diff 焦点模式

## Why

当前 Git 面板在改动文件列表中采用平铺列表，适合小规模改动，但在以下场景可读性不足：

- 多目录、多模块同时改动时，难以快速定位文件所属层级。
- 列表只展示文件名 + 截断路径，结构感弱，心智负担高。
- 用户期望类似 IDEA 的 Git 使用体验：树形结构快速定位 + 点击文件后聚焦查看该文件 Diff。

现有视图仍有价值（轻量、快速），因此不应替换，而应提供可切换模式。

## What Changes

### 1. 保留当前平铺视图（Flat）

- 现有 Git 文件列表行为保持不变。
- 作为默认视图继续可用，避免打断老用户习惯。

### 2. 新增树形视图（Tree）

- 在 Git 面板头部新增视图切换控件：`Flat | Tree`。
- Tree 模式按路径层级展示改动文件，支持目录折叠/展开。
- 文件项继续展示状态信息（A/M/D 等）与增删统计。

### 3. 新增单文件 Diff 焦点模式

- 在 Tree 模式点击文件时，右侧 Diff 区域聚焦该文件。
- Tree 模式下 Diff 区域仅展示当前选中文件，不再平铺所有文件 Diff。
- 无选中文件时展示空态引导（请先在树列表中选择文件）。

### 4. 文件级 Diff 头部控制（已落地）

- 将 Diff 视图控制收敛到“当前打开文件 header”：
- `split / unified` 视图切换按钮在文件 header 上显示。
- `全文查看 / 区域查看` 作用于当前文件（非全局面板）。
- `打开全文` 保留为文件全文编辑入口。
- 顶部独立操作条已移除，避免与文件级控制重复。

### 5. 全文查看数据源修正（已落地）

- `全文查看` 不再仅依赖 patch 周边上下文。
- 新增按文件获取 full-context diff 的数据链路，优先使用：
- `git diff HEAD --unified=999999 -- <file>`
- 在全文模式中展示包含未修改行的完整上下文。
- 按钮状态保留：`FULL / EMPTY / ERR / ...` 用于反馈数据命中状态。

### 6. 树形列表识别增强（已落地）

- 文件状态展示与识别优化：
- 采用 `(U)/(A)/(D)` 标记并配合文本色区分（未暂存/新增/删除）。
- 去除误导性的单选样式标记与旧的 `M` 前缀展示。

### 7. 全文模式浮动锚点（已落地）

- 仅在树形 + 当前文件 `全文查看` 时显示浮动锚点控件。
- 支持上/下一个改动跳转。
- 锚点分组规则：按“变更行是否连续”分组，连续记为同一锚点，行号跳变记为新锚点。
- 计数展示为 `当前/总数`，例如 `2/8`。

### 8. 状态持久化

- 记住每个 workspace 最近使用的 Git 列表视图模式（Flat 或 Tree）。
- 首版不强制持久化目录展开状态。

## Scope

### In Scope

- Git Diff 面板文件列表的展示模式切换（Flat/Tree）。
- Tree 节点交互与单文件 Diff 焦点行为。
- 视图模式本地持久化。

### Out of Scope

- 重写现有 Diff 渲染引擎。
- 引入三方大型树组件库（优先在现有实现上扩展）。
- 改动 Git 后端数据接口。

## UX Requirements

1. 用户能在 1 次点击内完成 Flat/Tree 切换。
2. Tree 视图中可直观看到文件所属目录结构。
3. 点击文件后默认只看该文件 Diff，减少视觉噪音。
4. 保证与当前批量 Stage/Unstage/Revert 行为兼容。

## Technical Notes (Current Code Alignment)

基于当前代码结构，改动主要集中在：

- `src/features/git/components/GitDiffPanel.tsx`
- `src/features/app/hooks/useGitPanelController.ts`
- 相关样式与本地设置持久化路径

当前已经有 `selectedDiffPath` 与 diff 选择链路，可复用为“单文件焦点”状态基础。

## Risks

1. 交互复杂度上升（两种视图 + 焦点模式）。
2. 大量文件时 Tree 展开性能与重渲染压力。
3. 现有多选批量操作在 Tree 模式下的行为一致性。

## Mitigations

1. 首版保持默认 Flat，不改变旧流程。
2. Tree 列表做懒展开与 memo 化。
3. 明确定义多选行为优先级：以“文件真实选择集合”为准，与视图形态解耦。

## Acceptance Criteria

1. Git 面板支持 Flat/Tree 双视图切换且可用。
2. Tree 视图可展示目录层级并支持折叠展开。
3. 点击 Tree 文件后，Diff 区域聚焦单文件展示。
4. 文件 header 提供 `split/unified` 与 `全文查看/区域查看` 控制。
5. 全文查看可展示包含未修改行的 full-context diff（按钮状态可见）。
6. 全文模式提供浮动锚点并可在改动点间跳转。
7. 现有 Stage/Unstage/Revert 基本流程不回归。
8. 视图模式在 workspace 维度可持久化。

## Rollout

- Phase 1: Flat/Tree 切换 + Tree 列表展示（已完成）
- Phase 2: 单文件 Diff 焦点 + 文件级 header 控制（已完成）
- Phase 3: 全文数据源修正 + 浮动锚点跳转（已完成）
- Phase 4: 键盘无障碍与快捷键补齐（进行中）
