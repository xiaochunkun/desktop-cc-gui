# Proposal: T1-4 Panel Lock Screen Overlay

## Why

CodeMoss 在共享办公场景下需要“临时离席保护”：

- 不中断后台会话执行
- 防止他人误触与旁观
- 提供可恢复访问入口

同时，锁屏页不应是空白，需要承载产品能力说明与运行态可视化。

## 目标与边界

- 目标：标题栏一键锁屏；锁屏后提供解锁面板与能力展示页；支持运行中会话可视化；会话完成时提供非自动消失提醒。
- 边界：仅做应用内 UI 层防护与本地密码校验，不引入系统级安全、账号鉴权或密码加密。

## 非目标

- 不实现系统级防护（防截屏、防调试等）。
- 不暂停/终止后台会话。
- 不做云端密码同步或多用户权限管理。

## What Changes

### 1) 标题栏锁屏入口

- 在主标题栏操作区增加锁头按钮（可着色高亮）。
- 点击进入锁屏遮罩，保持后台会话继续执行。

### 2) 锁屏遮罩与解锁流程

- 新增独立 `LockScreenOverlay` 页面（减少对既有主界面污染）。
- 遮罩提供密码输入与解锁，不再内置改密表单。
- 默认密码文件：`~/.codemoss/client/pwd.txt`，默认内容 `123456`。
- 若文件不存在：允许解锁，并自动创建默认密码文件，避免首次安装锁死。

### 3) 锁屏展示内容与交互

- 锁屏页采用 tab 结构（实时会话 / 能力图谱 / 交付流程 / 元素介绍）。
- 实时会话仅显示“运行中输出”（不展示历史），并保持流式刷新。
- 实时会话卡片高度自适应：1 条占满、2 条等分、更多按行分配并设置最小高度。
- 页面配色与系统主题联动（浅色/深色/暗淡），并接入 icon 资源。

### 4) 会话完成提醒气泡（右下）

- 主界面右下增加紧凑提醒气泡：手动关闭或跳转到对应会话，不自动消失。
- 同一 session 仅保留一条提醒：新完成覆盖旧提醒。
- 展示信息压缩为：完成时间、项目、最新会话片段。
- 与 update toast 自动避让，避免遮挡。

### 5) 完成事件检测稳健性

- 采用多信号判定会话完成并刷新提醒：
    - `isProcessing` true -> false
    - `lastDurationMs` 变化
    - `lastAgentMessage.timestamp` 推进
- 首次初始化不触发误报。

## 验收标准

- 点击锁头后可进入全屏锁屏遮罩，且后台会话不中断。
- 密码读取自 `~/.codemoss/client/pwd.txt`；文件缺失时可正常解锁并自动补默认文件。
- 锁屏页“实时会话”仅显示运行中的输出，且布局不溢出，卡片高度按数量自适应。
- 会话完成提醒在右下显示，不自动消失，可手动关闭或一键跳转会话。
- 同一 session 连续完成时提醒会被覆盖为最新内容（非首次旧内容）。
- 当 update toast 存在时，会话提醒自动上移避让。

## Capabilities

### New Capabilities

- `panel-lock-screen`: 一键锁屏、解锁与本地密码文件机制。
- `panel-lock-showcase`: 锁屏页能力展示 + 运行中会话可视化。
- `session-completion-bubble`: 会话完成后右下紧凑提醒（可跳转、可关闭、同 session 覆盖）。

### Modified Capabilities

- `main-header-actions`: 新增锁屏入口与交互态。

## Impact

- 前端：`App.tsx`、`MainHeader`、`LockScreenOverlay`、`ThreadCompletionBubble` 及相关样式/i18n。
- Tauri / 本地文件：锁屏密码读写接口对接 `~/.codemoss/client/pwd.txt`。
- 验证：已执行 `npm run typecheck` 与 `npm run build`，覆盖多轮迭代回归。
