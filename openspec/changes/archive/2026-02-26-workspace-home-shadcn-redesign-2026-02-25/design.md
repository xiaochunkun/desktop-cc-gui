## Context

Workspace Home 承载了 CodeMoss 的首要入口职责：用户进入工作区后在此选择引擎、启动/续接会话、进入 OpenSpec/Spec-kit
规范流，以及管理最近会话。当前实现的功能链路可用，但在生产场景下存在以下问题：

- 信息层级不稳定：首屏元素密度高，主次入口区分弱。
- 视觉一致性不足：深浅色主题切换后部分状态对比不足，风险态（删除确认）可辨识性不稳。
- 组件语义分散：虽然已有 shadcn/ui primitives，但首页局部仍偏“自定义样式堆叠”，后续演进成本偏高。

约束条件：

- 不能改变已有业务行为（引擎选择、会话启动、继续会话、批量删除确认逻辑）。
- 不能新增后端接口或改变 IPC 契约。
- 必须兼容当前 i18n 文案键与测试覆盖范围。

## Goals / Non-Goals

**Goals:**

- 使用 shadcn/ui 组件语义重构 Workspace Home 的信息架构与视觉层级。
- 在浅色/深色主题下维持一致的状态语义映射与可读性。
- 保证现有关键交互链路零语义回归。
- 为后续首页能力扩展提供稳定布局骨架和样式边界。

**Non-Goals:**

- 不改变 Spec Hub 工作台（三栏）设计。
- 不引入新的业务动作或新的后端状态。
- 不做全局主题系统重写，仅消费既有 token。

## Decisions

### Decision 1: 采用“结构重排 + 行为复用”而非“行为重写”

- 方案 A：在现有结构上仅做样式微调。
- 方案 B：重写组件结构，同时复用现有 handlers/state（采用）。
- 方案 C：重写结构与状态机。

选择 B 的原因：可以解决信息架构问题，同时最大化复用既有行为与测试资产，回归风险可控。

### Decision 2: 深浅色统一通过语义 token 驱动，不做硬编码主题分支

- 方案 A：light/dark 分开写大量样式分支。
- 方案 B：以语义变量驱动（surface/border/text/status）并只在必要处补差异（采用）。

选择 B 的原因：减少主题漂移与后续维护成本，保证状态色语义在两种主题中等价。

### Decision 3: 首页采用三段式布局骨架，卡片化承载主入口

布局骨架：

1. Hero：工作区身份 + 路径 + 引擎选择 + 主 CTA。
2. Guides：OpenSpec/Spec-kit 入口 + 通用引导动作。
3. Recents：最近会话列表 + 批量管理条。

该骨架直接映射用户主任务顺序（启动 -> 选择流程 -> 回到上下文），降低扫描成本。

### Decision 4: 最近会话管理维持“显式管理态”与“双击确认删除”状态机

伪流程：

- 默认态：点击会话=进入会话。
- 管理态：点击会话=切换选中。
- 删除按钮首次点击：进入 armed。
- 再次确认：执行删除；失败仅保留失败项选中。

该状态机沿用现有能力，仅增强视觉表达，不改变语义。

### Decision 5: 以 shadcn/ui primitives 作为主承载

- `Card`：区域容器与层级分组。
- `Button`：主次操作与风险操作。
- `Badge`：上下文身份标签（工作区类型/分支）。
- `Select`：引擎选择。

目标是让首页视觉与项目其余 shadcn 区域保持同源风格，避免“单页异构”。

## Risks / Trade-offs

- [Risk] 结构调整导致测试选择器或可访问名称波动。
  -> Mitigation: 保留文案 key 和核心按钮 aria 名称；必要时仅最小修正脆弱断言。

- [Risk] 深色模式下危险态（删除确认）与普通态对比不足。
  -> Mitigation: 为风险态定义独立 token 组合（边框/背景/文字）并做深浅色双态检查。

- [Risk] 过度追求视觉改造引入行为回归。
  -> Mitigation: 明确“行为复用优先”，不修改现有 handler 输入输出契约。

- [Risk] CSS 重写范围过大影响其他页面。
  -> Mitigation: 严格限定在 `workspace-home*` 命名空间，不触碰全局样式规则。

## Migration Plan

1. 基线阶段

- 锁定现有 Workspace Home 行为测试作为回归护栏。

2. 结构阶段

- 重排 `WorkspaceHome.tsx` 与 `WorkspaceHomeSpecModule.tsx` 的布局层级。
- 复用原有状态与事件处理函数。

3. 样式阶段

- 重写 `workspace-home.css`，建立三段式布局与主题语义映射。
- 验证浅色/深色模式可读性和交互反馈。

4. 验证阶段

- 运行 Workspace Home 相关测试，人工检查关键交互链路。

Rollback Strategy:

- 保留单次提交边界（结构与样式分离）；若出现回归，可先回滚样式层到上一个稳定版本，再逐步恢复结构优化。
