## Context

当前 Codex 模式在对话区同时存在多种文件变更呈现路径：`File changes` 卡片、`批量编辑文件`卡片、流式文本中的文件链接。
这些路径由不同渲染链路驱动，导致交互协议不一致（有的可点、有的不可点、有的点击崩溃），并出现弹层层级冲突和信息重复展示。

约束与相关方：

- 约束：不新增后端接口，不引入新依赖；保持现有会话与操作数据结构兼容。
- 相关方：前端对话区组件、Git 风格 diff 预览组件、状态聚合逻辑、测试基线。

## Goals / Non-Goals

**Goals:**

- 统一文件点击行为：所有文件引用点击后走同一“文件详情弹窗”链路。
- 修复稳定性：流式文本中的文件点击不得触发崩溃。
- 统一弹层策略：`Plan` 与文件详情弹层不被卡片容器遮挡。
- 明确展示职责：`File changes` 为变更详情主视图，`批量编辑文件`为摘要与操作入口。
- 补全可读性：`File changes` 支持单文件 `+n/-m`、总计 `+total/-total`、`A/M/D` 状态标识。
- 控制变更范围：以最小改动修复现有行为，不改变业务语义。

**Non-Goals:**

- 不重构消息存储模型或会话协议。
- 不重写整套 diff 渲染器。
- 不新增复杂权限、协作或远程文件访问能力。

## Decisions

### 决策 1：建立统一 File Reference Action 协议

- 方案：引入统一入口（例如 `openFileReference(payload)`），覆盖三类触发源：`File changes` 文件项、`批量编辑文件`文件项、流式文本文件链接。
- 选择理由：避免多处各自实现点击逻辑导致行为漂移与崩溃分叉。
- 备选方案：
    - 备选 A：每个卡片保留独立 click handler。
        - 未选原因：修复成本分散，后续新增入口容易漏修。

### 决策 2：弹层统一走顶层 Portal + 层级 Token

- 方案：文件详情弹窗与 Plan 弹窗统一挂载到应用顶层 Portal，使用统一 z-index token（例如 modal/popover 分层表）。
- 选择理由：一次性解决被卡片容器 stacking context 遮挡问题。
- 备选方案：
    - 备选 A：逐个组件局部调高 `z-index`。
        - 未选原因：短期可见，长期易互相覆盖，回归风险高。

### 决策 3：定义“主详情 + 摘要”双卡职责

- 方案：
    - `File changes`：唯一详情区（文件列表、状态、统计、点击弹窗入口）。
    - `批量编辑文件`：仅保留摘要信息与操作入口（如 Plan），不重复铺开同级详情。
- 选择理由：减少语义重复，降低阅读负担。
- 备选方案：
    - 备选 A：两张卡都展示完整文件详情。
        - 未选原因：重复信息增加噪音，且两个视图容易不一致。

### 决策 4：统计与状态标识由统一映射层产出

- 方案：在前端增加“变更展示映射层”，将原始 diff 数据映射为：
    - `file.status`: `A/M/D`
    - `file.additions`, `file.deletions`
    - `summary.totalAdditions`, `summary.totalDeletions`
- 选择理由：把显示规则从组件中抽离，避免重复计算与格式差异。
- 备选方案：
    - 备选 A：每个组件内自行计算统计。
        - 未选原因：重复逻辑难统一，测试覆盖成本高。

### 决策 5：点击异常降级兜底，禁止崩溃

- 方案：文件点击链路增加输入校验与异常兜底：
    - payload 不完整时提示“无法定位文件”而非抛出未捕获异常；
    - 弹窗打开失败时返回可恢复提示，不影响会话继续。
- 选择理由：优先保障主流程可用性。
- 备选方案：
    - 备选 A：保留现状，依赖调用方保证参数完整。
        - 未选原因：与现有偶发崩溃事实冲突。

## Risks / Trade-offs

- [Risk] 多入口统一后，历史入口可能被遗漏 → Mitigation：建立“触发源清单”并用测试逐项覆盖。
- [Risk] 层级统一可能影响现有其他弹层顺序 → Mitigation：定义分层 token 表并回归关键弹层场景。
- [Risk] 去重策略误判导致信息缺失 → Mitigation：先采用“同 operationId + 同文件集合”保守规则。
- [Risk] 统计字段来源不一致导致数字漂移 → Mitigation：仅允许映射层产出展示字段，组件禁止二次计算。

## Migration Plan

1. 第 1 步：新增统一 File Reference Action 与参数类型，接入三类触发源。
2. 第 2 步：文件详情弹窗与 Plan 弹窗迁移至顶层 Portal，切换统一层级 token。
3. 第 3 步：落地“主详情 + 摘要”职责拆分，去除重复详情展示。
4. 第 4 步：接入统计/状态映射层，补全 `A/M/D` 与 `+/-` 单文件及总计显示。
5. 第 5 步：补齐测试（点击稳定性、弹层层级、去重规则、统计正确性）。

回滚策略：

- 保留旧渲染分支开关（按组件级 feature flag 或 guard），出现回归时可临时回退到旧展示路径。
- 回滚不涉及数据迁移，仅 UI/交互层切换。

## Open Questions

- 流式文本文件链接的最小 payload 规范是否已经固定（是否总能提供 repoRoot + relativePath）？
- 文件详情弹窗默认展示模式是否统一为 diff 视图，还是按来源决定（diff/原文）？
- 去重规则在跨消息聚合场景下是否需要额外时间窗口限制？
