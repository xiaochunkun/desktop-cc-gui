## Why

当前客户端右侧文件树直接平铺工作区一级目录与文件，缺少显式“工作区根节点”，导致层级语义不清晰，用户难以快速判断当前浏览范围。随着文件树能力持续扩展（渐进加载、搜索、运行入口等），现在补齐根节点契约可以降低后续交互演进成本并统一视觉结构。

## 目标与边界

- 目标：在右侧文件树顶部新增可见的工作区根节点（例如 `SPRINGBOOT-DEMO`），其下承载现有文件与目录树。
- 目标：根节点支持折叠/展开，默认展开，与现有目录节点交互语义一致。
- 目标：在根节点绑定并复用现有目录右键菜单能力（如新建文件、创建副本、复制路径、在访达中显示、移到废纸篓）。
- 目标：将“筛选文件和文件夹”搜索输入框上移到顶部工具行，与根节点标题区形成同一行布局（不再独占下一行）。
- 目标：根节点所在顶部行采用吸顶（sticky）行为，文件列表滚动时根节点与顶部工具行保持固定，不随内容滚动离开视口。
- 边界：本提案只定义“根节点显示与交互契约”，不改变业务文件读写能力。
- 边界：本提案不引入多根工作区（multi-root）模型。

## 非目标

- 不重构文件树搜索算法与索引结构。
- 不改变特殊目录渐进加载策略（`node_modules`、`target` 等）。
- 不修改文件打开、多 Tab、Diff/Preview 等既有下游功能语义。
- 不新增搜索语法或过滤规则，仅调整搜索框位置与布局编排。

## What Changes

- 新增“工作区根节点”显示能力：文件树最上层固定渲染当前工作区名作为单一根节点。
- 调整初始树数据结构契约：现有顶层节点迁移为“根节点的 direct children”。
- 新增根节点折叠行为契约：根节点折叠时隐藏整个子树，展开后恢复上次展开状态。
- 新增根节点右键菜单契约：根节点触发上下文菜单时，复用现有目录菜单入口与执行链路，不新增分叉菜单实现。
- 新增根节点状态显示约定：在根节点行保留/兼容现有右侧操作入口（如刷新、创建等）的位置策略。
- 调整搜索框布局契约：将“筛选文件和文件夹”输入框从独立第二行迁移到顶部行，并保证与根节点标题及右侧操作区不重叠。
- 新增滚动行为契约：根节点与顶部工具行作为同一吸顶区域，滚动仅作用于文件列表内容区。
- 保持兼容：现有目录/文件节点的选择、展开、打开与上下文菜单行为不回退。

## 方案对比（至少两个）

### 方案 A：前端视觉伪根（不改后端数据契约）

- 做法：前端在渲染层注入一个虚拟根节点，把当前顶层列表包裹进去。
- 优点：实现快，后端无改动。
- 缺点：数据语义与 UI 语义分离，后续功能（路径映射、状态持久化、测试）复杂度上升。

### 方案 B：端到端真实根节点（调整树数据契约）

- 做法：在文件树数据模型中引入真实根节点，前后端统一按“root -> children”处理。
- 优点：模型一致，状态管理更稳定，测试与后续扩展（如根节点统计/权限提示）更自然。
- 缺点：改动面略大，需要同步更新部分适配层与测试。

### 取舍

采用 **方案 B（端到端真实根节点）**。该方案一次性消除语义分裂，符合后续文件树能力持续演进的长期维护收益。

## Capabilities

### New Capabilities
- `workspace-filetree-root-node`: 定义右侧文件树的单一工作区根节点展示、折叠展开与状态保持契约。

### Modified Capabilities
- `workspace-filetree-special-directory-loading`: 顶层 listing 语义从“工作区顶层节点列表”调整为“根节点 direct children 列表”，并保持特殊目录渐进加载契约不变。
- `workspace-sidebar-visual-harmony`: 新增根节点后，侧边栏层级与间距/图标语义需保持一致性与可读性。
- `filetree-multitab-open`: 根节点引入后，右键菜单与文件打开链路需继续保持既有行为一致，不影响文件多 Tab 打开契约。

## 验收标准

- 打开任意工作区后，文件树顶部必须显示且仅显示一个工作区根节点，名称与当前工作区一致。
- 根节点默认展开，能手动折叠/展开；折叠后子树全部隐藏，展开后恢复。
- 在根节点触发右键菜单时，菜单项与既有目录菜单保持一致（至少包含：新建文件、创建副本、复制路径、在访达中显示、移到废纸篓）。
- 根节点右键菜单各动作需复用现有命令执行链路，行为与在普通目录触发时一致。
- 搜索输入框需位于顶部工具行，与根节点标题区同层显示；不得遮挡右侧文件计数与操作按钮。
- 在常见宽度下（含你截图所示布局）搜索输入框内容可见且可输入，placeholder 不被裁剪。
- 文件列表纵向滚动时，根节点与顶部工具行必须保持吸顶固定；滚动条只驱动下方文件列表区域。
- 吸顶区域层级需高于列表内容，滚动过程中不得出现内容覆盖根节点行或操作按钮点击失效。
- 根节点下的文件与目录顺序、图标、展开行为与当前实现保持一致（除新增层级外无回退）。
- 特殊目录（如 `node_modules`、`target`）仍按既有渐进加载规则工作。
- 现有文件打开、多 Tab、Diff/Preview、上下文菜单链路回归通过。
