# Proposal: Git Panel Tree View Accessibility & Test Follow-up

## Why

`git-panel-tree-view-single-diff` 已完成核心体验（Tree 视图、单文件焦点、全文模式与锚点跳转），但仍有 7 个遗留项未纳入交付闭环：

- 可用性：键盘导航、视图切换快捷键、基础无障碍属性。
- 质量保障：树构建单测、视图焦点行为组件测试、flat 模式回归、大变更集性能手工验证。

这些项不影响当前功能可用，但会影响可访问性、可维护性和后续迭代信心。

## What Changes

1. 为 Tree 文件列表补齐键盘交互（方向键/Enter），保证不依赖鼠标也可完成定位与打开。
2. 增加 Git diff 列表视图切换快捷键（需先完成冲突扫描并确定最终组合）。
3. 补齐基础 a11y 语义（`aria-label`、`aria-expanded`、焦点可见性一致性）。
4. 建立测试覆盖：
    - 树构建逻辑单元测试；
    - 视图切换与单文件焦点组件测试；
    - flat 模式核心链路回归测试。
5. 完成大变更集（高文件数）下的手工性能验证记录。

## Scope

### In Scope

- Git Tree 列表键盘可达性与文件选择行为。
- Git Flat/Tree 切换快捷键及冲突规避。
- 与本能力直接相关的 a11y 属性增强。
- 与本能力直接相关的单测/组件测试/回归测试。
- 手工性能验证与结果记录。

### Out of Scope

- 重写 Diff 渲染器。
- 新增大型可访问性框架依赖。
- 与 Git 面板无关模块的全局快捷键重构。

## Acceptance Criteria

1. Tree 模式支持 `↑/↓` 选择文件项，`←/→` 折叠展开目录，`Enter` 打开/聚焦文件。
2. 视图切换快捷键可用，且与已有全局快捷键无冲突。
3. 关键交互节点具备基础 a11y 属性并通过基本键盘走查。
4. 新增测试覆盖上述行为，flat 模式核心流程有回归保障。
5. 输出大变更集手工性能验证结论（包含场景与观察结果）。
