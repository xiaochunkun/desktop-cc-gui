## Why

当前多文件 Tab 已可并行打开，但 Chat 与当前选中文件仍是“弱关联”：

- 用户在文件页提问时，不清楚本次问题是否携带了当前文件上下文。
- 发送后虽然可回到 chat，但消息内容与当前文件缺乏显式锚点。

这会导致“我在问哪一个文件”的歧义，影响 AI 回复准确性与可追溯性。

## What Changes

- 在 Composer 中新增“当前文件关联条”，显示 active file tab。
- 提供文件路径注入开关：发送时自动注入 `@file \`path\``（可关闭）。
- 发送/排队在 editor 场景后自动切回 chat，确保可见反馈闭环。
- 优化 Tab 选中态视觉区分。

## 目标与边界

- 目标
    - 让用户明确知道“当前提问关联的是哪个文件”。
    - 降低文件上下文遗漏导致的答非所问。
    - 保持发送链路不改协议。
- 边界
    - 本轮只做“文件路径级关联”，不做行级/片段级引用。
    - 不改后端消息协议，不新增 Tauri command。

## 非目标

- 不实现自动抽取选中代码片段。
- 不实现多文件批量注入。
- 不实现会话级长期文件 pin 规则。

## 技术方案对比

### 方案 A：发送前弹窗二次确认文件关联

- 优点：显式、可控。
- 缺点：每次发送都有额外操作，打断流。

### 方案 B：Composer 内联关联条 + 一键开关（推荐）

- 优点：低打断、状态持续可见、实现最小。
- 缺点：需要用户理解开关状态。

### 取舍

采用 **方案 B**，先打通轻量可用闭环，再迭代行级关联。

## Capabilities

### New Capabilities

- `composer-active-file-reference`: Composer 显示当前文件关联并支持路径注入开关。

### Modified Capabilities

- `filetree-multitab-open`: 强化 Tab 选中态视觉区分。

## 验收标准

- Composer 可显示 active file 标识（文件名可见，完整路径可悬浮查看）。
- 路径注入开关开启时，发送消息包含 `@file \`path\`` 前缀。
- 路径注入开关关闭时，发送消息不附加文件路径。
- editor 场景发送后，界面切回 chat 显示消息反馈。
